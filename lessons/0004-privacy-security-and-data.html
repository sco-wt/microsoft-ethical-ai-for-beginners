Lesson 0004

Privacy, Security, and Data

Course Home

Goal: trace where sensitive data enters, how it is transformed, and how it might leak, persist, or be misused.

The Core Idea

Privacy is not a checkbox at the end. In AI systems, data can appear in prompts, training sets, logs, embeddings, model outputs, analytics, and human review queues.

The Data Path

Collect What data is gathered, and is it necessary?
Store Where does data persist, and for how long?
Use Who or what can access it during inference or review?
Expose Could outputs reveal private facts or sensitive attributes?

Quick Check

1. "Do we need this field at all?" is mainly which privacy habit?

2. "How long do prompts stay in logs?" is mainly about what?

3. "Could the answer reveal private details?" checks what?

Practice Prompt

Draw the data path for a chatbot: user input, model call, logs, analytics, human review, and output. Mark every place sensitive data could persist.
